# PRD: Crypto研究平台信息架构重构

**需求方**: 星哥  
**提出时间**: 2026-04-22  
**优先级**: P0  
**状态**: Draft

## 背景
当前 `singclaw.xyz/crypto/` 下有6个平级页面，信息架构扁平，用户无法直观理解"信号→回测→模拟→复盘"的完整交易研究闭环。

## 目标
1. 建立一二级菜单结构，按交易研究生命周期组织内容
2. 让数据流可视化：信号识别 → 回测验证 → 模拟交易 → 复盘调整
3. 为后续新增模块预留扩展空间

## 需求详情

### IA 方案 A（推荐）
```
Crypto Alpha 📊 (总入口/仪表盘)
├── 信号发现 📡
│   ├── 信号追踪 (signal-tracker.html)
│   ├── 观察池 (watchlist.html)
│   └── BN Alpha (bn-alpha.html)
├── 研究验证 🔬
│   ├── 回测中心 (backtest.html)
│   └── 模拟交易 (paper-trade.html)
└── 迭代复盘 📋
    └── 每日复盘 (review.html)
```

### IA 方案 B（紧凑版）
```
Crypto Alpha 📊
├── 信号中心 📡
│   ├── 信号追踪 + 观察池 + BN Alpha (合并为一页三栏)
├── 回测中心 🧪
│   ├── 回测 + 模拟交易
└── 每日复盘 📋
```

### 交互要求
1. 顶部一级Tab切换（信号发现 / 研究验证 / 迭代复盘）
2. 二级Tab或侧边栏导航
3. 移动端汉堡菜单适配
4. 数据流可视化图表（可选）

### 技术实现
- 保持现有静态HTML架构
- 可考虑引入轻量级JS路由或iframe嵌入
- 或保持独立页面，统一导航组件

## 验收标准
- [ ] 一二级导航清晰，用户3秒内理解信息结构
- [ ] 所有现有页面内容无损迁移
- [ ] 移动端导航可用
- [ ] 数据流串联展示（信号→回测→模拟→复盘）

## 排期
- 设计: 1天
- 开发: 2天
- 测试部署: 0.5天
- **总计**: 3.5天
