# PRD: Crypto研究平台信息架构重构
> **需求方**: 星哥 | **日期**: 2026-04-22
> **优先级**: P0 | **状态**: Draft

## 1. 背景与问题

### 1.1 现状
当前 `singclaw.xyz/crypto/` 下有6个平级页面：
- index.html (5层分析框架)
- bn-alpha.html (BN Alpha统计)
- watchlist.html (观察池)
- signal-tracker.html (信号追踪)
- backtest.html (回测中心) - 新建
- paper-trade.html (模拟交易) - 新建
- review.html (每日复盘) - 新建

### 1.2 问题
- 页面太多太散，用户不知道从哪里开始
- 缺少信息层级，所有页面平级展示
- 数据流不清晰：信号→回测→模拟→复盘 没有串联

## 2. 目标

### 2.1 产品目标
- 将孤立页面整合为**有层级的研究平台**
- 清晰展示**交易研究闭环**：信号识别 → 回测验证 → 模拟交易 → 复盘调整
- 提升用户导航效率

### 2.2 用户故事
- 作为交易者，我想先看到今日信号，然后查看历史回测表现，最后模拟交易验证
- 作为研究者，我想按场景分类查看信号，对比不同策略的回测结果

## 4. 功能需求

### 4.1 信号中心
- [ ] 信号列表（Score、价格、4h/8h涨幅）
- [ ] 观察池（OI场景分类、标的监控）
- [ ] 信号→观察池联动（高分信号自动加入观察池）

### 4.2 研究分析
- [ ] 回测中心（300次OI回测统计）
- [ ] BN Alpha（合约vs现货超额分析）
- [ ] 回测→信号联动（展示历史信号的实际表现）

### 4.3 模拟交易
- [ ] 基于信号自动开仓（Score≥70）
- [ ] 虚拟持仓追踪
- [ ] 盈亏统计

### 4.4 每日复盘
- [ ] 信号表现回顾
- [ ] 参数优化建议
- [ ] 策略迭代记录

## 5. 数据流设计

```
数据治理管线 (每15min)
    ↓
信号识别 (Score≥70)
    ↓
信号追踪页面 ←→ 观察池
    ↓
回测中心 (历史验证)
    ↓
模拟交易 (虚拟开仓)
    ↓
每日复盘 (参数优化)
    ↓
Score算法迭代
```

## 6. 技术要求

### 6.1 前端
- 一级导航页面需要卡片式布局，展示各模块入口
- 二级页面保持现有样式，更新导航栏
- 移动端适配

### 6.2 数据
- 信号数据自动从 `data-governance/data/analysis/signals/` 读取
- 回测数据从 `crypto/oi_backtest_result.json` 读取
- 模拟交易数据需要新建存储
