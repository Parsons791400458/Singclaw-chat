## ⚠️ 仍在 backlog
- **工具调用未接 fast-path**：模型尝试 `web_search` / `coin_price` 都报"工具暂不可用"，fallback 到文本回复。S9.5.3 真 OpenAI/Anthropic function calling 是真修。
- **adapter 没接 systemd 守护**：adapter 进程是 nohup，重启 host 会再失联。**建议**：加 systemd unit 或 pm2 ecosystem 入口，避免下次重启 H Sing 又卡。
- **user turn 在 abort 后不持久化**：客户端断流（max-time < 回复时间），DB 只看到 agent turn。无影响（实际用户等回完），但 edge case。
- **singclaw-v2 旧仓库归档**：源码未删（不动 H Sing 资产），但 production 不再 deploy。Vercel 项目也未重部署。

## 📐 治理建议（PMO 摘记）
1. **build/deploy 链路缺验证门**：源码 commit 没问题，但 build 没刷新，pm2 跑的是 7h 前的 build。这次差点又翻车。
   - 建议：singclaw-dynamic ecosystem 配置里加 `post-deploy: curl /api/mvp-proxy | grep S9.5.2 || exit 1`
2. **adapter 守护缺失**：FastAPI 是 nohup，不在 pm2 不在 systemd。
   - 建议：挪进 pm2（`pm2 start 'uvicorn adapter.server:app --port 8000' --name singclaw-mvp-adapter`）或加 systemd unit。
3. **mvp-proxy stage 字面值误导**：`S9.5.2-C-BFF-mvp-proxy` 是 const，但老 build 跑出来还是旧字面值。生产环境从 stage 字符判断"哪个版本"已不可信。
   - 建议：stage 改成 `git rev-parse HEAD` 或加 `build_hash` 字段。
